Understanding GPT from Scratch

A complete walkthrough of a working GPT in Java — from zero math background to full understanding.

What is this? In February 2026, Andrej Karpathy released microGPT — a complete GPT (the same architecture behind ChatGPT) in just 200 lines of Python, with zero dependencies. A developer named dev4any1 ported it to Java in ~300 lines. This course takes that Java code and explains every single line — even if you've never heard of a vector, a gradient, or a probability distribution.

Who is this for?

You are a developer (you know Java, loops, classes, arrays), but you don't know:

  • What a vector is (beyond "an array of numbers")
  • What an embedding means
  • What probabilities, Gaussian distributions, softmax do
  • What a gradient or derivative is
  • How a neural network learns
  • What attention means in "Attention Is All You Need"

By the end, you will understand every concept and every line of the 300-line Java GPT.

How to use this course

100% Offline This entire site works without internet. Open index.html in your browser — on a plane, in a train, wherever. All JavaScript, charts, and interactive demos are self-contained.

Read the chapters in order. Each one builds on the previous. Interactive demos let you experiment with the concepts — drag vectors, adjust parameters, see how things change in real time.

The Code

This is the entire program we're going to understand — a working GPT in ~300 lines of Java. It trains on a dataset of 32,000 human names and learns to invent new ones. The same architecture powers ChatGPT, just scaled up by a factor of 1,000,000x.

This micro GPT

  • 1 transformer layer
  • 4 attention heads
  • 16-dimensional embeddings
  • 4,192 parameters
  • Trains in minutes on CPU
  • Learns to generate names

ChatGPT (GPT-4) (estimated*)

  • ~120 transformer layers
  • ~96 attention heads
  • ~12,000-dimensional embeddings
  • ~1.8 trillion parameters
  • Trains for months on thousands of GPUs
  • Learns human language
Same recipe, different scale (*GPT-4's exact architecture has not been published by OpenAI; figures above are widely cited estimates.) Every concept in the micro version (embeddings, attention, softmax, backpropagation, Adam) is exactly the same in the full version. The difference is only the numbers — more layers, more heads, more dimensions, more data, more compute. Understanding the micro version gives you the complete picture.
