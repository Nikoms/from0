Glossary

Reference Every term used in the course, in plain language, with a link to the chapter that explains it in full.

Jargon is the biggest barrier to understanding this stuff — not the ideas themselves. Each entry below is written for someone who knows how to program but has never touched the math or the machine-learning vocabulary. Tap a chapter chip to jump to the deep dive.

Activation function Ch06

A simple function applied to a neuron's output to introduce non-linearity, so the network can learn curved/complex patterns rather than only straight-line relationships. Ours uses ReLU.

Adam optimizer Ch09

The update rule that adjusts parameters during training. It improves on plain gradient descent by keeping a running average of each parameter's gradient (momentum) and of its squared gradient (to size each step adaptively).

Attention / Self-attention Ch07

The mechanism that lets each token look at other tokens and pull in information from them, weighted by relevance. “Self” means a sequence attends to itself. It's how the model uses context.

Autograd / Automatic differentiation Ch05

A technique that automatically computes gradients (derivatives) of a result with respect to every input, by recording each arithmetic operation. Our Value record is a tiny autograd engine.

Backpropagation Ch09

Running the chain rule backwards through the computation graph to fill in every parameter's gradient. Triggered by one call to loss.backward().

Batch / Mini-batch

A group of training examples processed together before updating parameters. Larger batches give steadier gradients. Our minimal version uses one name per step (a batch of one).

BOS (Beginning of Sequence) Ch01

A special token (id 26 in our model) that marks where a name starts and ends — a bookend for the machine. When the model generates a BOS, it's signalling “the name is finished.”

Causal attention / Causal mask Ch07

A rule that each position may attend only to earlier positions and itself, never the future. It's what lets the model predict the next token without cheating by peeking ahead. Our code enforces it naturally by only caching keys/values up to the current position.

Chain rule Ch05

The calculus rule for differentiating nested functions: multiply the local derivatives along the chain. It's the mathematical backbone of backpropagation.

Computation graph Ch05

The network of operations linking inputs to a final result, recorded as each operation runs. Walking it backwards yields the gradients. Built automatically by our Value operations.

Cross-entropy loss Ch09

The loss used here: −log(probability the model gave the correct token). Zero when perfectly confident and right; large when confidently wrong. Drives all the learning.

Derivative / Gradient Ch05

A derivative measures how fast one quantity changes as another wiggles. A gradient is the collection of such slopes for all parameters — it points uphill on the loss, so training steps the opposite way.

Dimensionality reduction Ch02

Squeezing high-dimensional data down to a few dimensions (e.g. 2) so it can be visualised or simplified, while preserving as much structure as possible. PCA is one method.

Dot product Ch02

Multiply two vectors element-by-element and sum the results, giving a single number that measures how aligned they are. The basic operation inside linear() and attention scoring.

Embedding Ch03

A learned vector of numbers that represents a token. Similar tokens end up with similar vectors. Our wte table holds one 16-number embedding per character.

Epoch

One full pass over the entire training dataset. With 32,033 names and 1,000 steps of one name each, our run doesn't even finish a single epoch — yet it still learns.

exp() / Exponential function Ch04

e raised to a power. It turns any number into a positive one and grows very fast. Softmax uses it to convert scores into positive quantities before normalising.

Forward pass

Running input through the model to produce an output (the logits/prediction), building the computation graph along the way. The opposite direction from backpropagation. In our code, the gpt() function.

GPT (Generative Pre-trained Transformer) Ch01

A model that generates text one token at a time, was trained ahead of time on lots of examples, and uses the Transformer architecture. ChatGPT and our 4,192-number name generator are the same idea at different scales.

Gradient descent Ch09

The core learning method: nudge each parameter a small step opposite to its gradient to reduce the loss, and repeat. Like rolling a marble downhill.

Head (attention head) Ch07

One of several parallel attention computations. Each head can focus on a different kind of relationship. Ours has 4 heads, each working on 4 of the 16 embedding dimensions.

Hyperparameter

A setting we choose rather than learn — e.g. number of layers, embedding size, learning rate, number of steps. Contrast with parameters, which the model learns.

Inference Ch01

Using a trained model to produce output — here, generating new names. No learning happens during inference; the parameters are frozen.

KV Cache

An efficiency trick: store the key and value vectors already computed for earlier positions so they don't have to be recomputed as generation proceeds. Our keys[]/values[] lists are exactly this cache.

Layer Ch06

One stage of processing in the network. A transformer layer bundles an attention block and an MLP block. Our model has a single layer (nLayer = 1); big models stack dozens.

Learning rate Ch09

How big a step each update takes. Too small = painfully slow; too large = overshoots and diverges. Ours starts at 0.01 and decays linearly to 0.

Linear transformation Ch06

Multiplying an input vector by a weight matrix to produce a new vector — a weighted mix of the inputs. In general, linear layers can include a bias term, but ours omit it for simplicity. Implemented by linear(); the most common operation in the model.

log() / Logarithm Ch04

The inverse of exponentiation. The natural log (base e) of a probability between 0 and 1 is negative and dives toward −∞ near 0 — which is why −log makes a clean loss.

Logits Ch04

The raw, unnormalised scores the model outputs for each possible next token, before softmax turns them into probabilities. Our model produces 27 logits per step.

Loss function Ch09

A single number measuring how wrong the model's predictions are; smaller is better. Training works by pushing it down. Ours is cross-entropy loss, averaged over a name.

MLP (Multi-Layer Perceptron) Ch06

A small feed-forward network: linear → activation (ReLU) → linear. In a transformer it processes each position after attention, expanding to 4× width and back.

Multi-head attention Ch07

Running several attention heads in parallel and concatenating their results, so the model can attend to different things at once. Ours runs 4 heads per layer.

Next-token prediction Ch01

The fundamental task a GPT performs: given the text so far, predict the next token. Doing this well forces the model to absorb real patterns. Our model predicts the next character.

One-hot encoding Ch03

Representing a token as a vector that is all zeros except a single 1 at the token's index. Multiplying a one-hot by the embedding table simply selects that token's row — which is what an embedding lookup does.

Parameter Ch01

One of the numbers the model learns (also called a weight). Our model has exactly 4,192; GPT-4 has ~1.8 trillion. The model's “knowledge” lives entirely in these values.

PCA (Principal Component Analysis) Ch02

A classic technique that finds the directions of greatest variation in data, used to reduce many dimensions down to a few for visualisation. A way to peek at high-dimensional vectors.

Position embedding Ch03

A learned vector added to a token's embedding to tell the model where in the sequence the token sits, since attention alone has no inherent sense of order. Our wpe table.

Probability distribution Ch04

A set of non-negative numbers that sum to 1, giving the chance of each outcome. The model's prediction is a distribution over the 27 possible next tokens.

Query, Key, Value Ch07

Three vectors derived from each token. A query asks “what am I looking for?”, keys advertise “what I offer”, and values carry the actual content. Query·Key scores decide how much of each value to pull in.

ReLU (Rectified Linear Unit) Ch06

The activation function max(0, x): pass positives through, clamp negatives to zero. Simple, fast, and enough to give the network non-linear power.

Residual connection Ch06

Adding a block's input back onto its output (x + block(x)). It keeps gradients flowing and lets each block learn a small adjustment rather than a full rewrite. Wrapped around both attention and the MLP.

RMSNorm (Root Mean Square Normalization) Ch06

A normalization that rescales a vector so its root-mean-square magnitude is about 1, keeping numbers stable as they flow through layers. Implemented in rmsnorm().

Softmax Ch04

The function that converts a list of logits into a probability distribution: exponentiate each, then divide by the sum. Used inside attention and to produce the final next-token probabilities.

Temperature Ch04

A knob applied during generation that divides the logits. Below 1 (ours is 0.5) it sharpens toward confident, “safe” picks; above 1 it flattens toward randomness and surprise.

Token / Tokenizer Ch01

A token is the smallest chunk of text the model reads/writes — a single character in our model, a word-fragment in ChatGPT. The tokenizer converts text to token ids and back.

Topological sort Ch05

An ordering of the computation graph's nodes such that every node comes after all the nodes it depends on. backward() reverses this order so gradients are computed in the correct sequence.

Transformer Ch08

The neural-network architecture behind GPTs, introduced in 2017's “Attention Is All You Need.” It stacks attention + MLP blocks with residuals and normalization. Our gpt() is one transformer block.

Vector Ch02

An ordered list of numbers — the fundamental data type of machine learning. A token's embedding is a 16-dimensional vector; everything the model does is arithmetic on vectors.

Vocabulary Ch01

The complete set of tokens a model knows. Ours is 27: the 26 lowercase letters plus BOS. GPT-4's vocabulary is ~100,000 word-fragments.

Weight matrix Ch06

A grid of parameters that transforms one vector into another via linear(). Each named entry in our sd map (e.g. attn_wq, mlp_fc1) is a weight matrix.
