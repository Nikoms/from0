Neural Network Layers

A neural network is a stack of simple functions called layers. Three of them carry almost all the weight in microGPT: the linear transform, ReLU, and RMSNorm. Master these and the transformer stops being magic.

Where we are You now understand Value — a smart number that tracks gradients. Every number in this chapter is a Value, so everything we build is automatically differentiable. We never have to think about derivatives again; we just compose layers, and the engine handles the learning.

1. What is a "layer"?

Strip away the intimidating vocabulary and a layer is just this:

Definition A layer is a function that takes a vector in and produces a vector out. Vector in → vector out. That's all.

Recall from the Vectors chapter that a vector is just a fixed-length list of numbers — in microGPT, usually 16 of them (nEmbd = 16). A layer reads those 16 numbers and produces a new list of numbers. Stack several layers and the output of one becomes the input of the next, like a pipeline:

[16 numbers] → layer → [16 numbers] → layer → [16 numbers] → ...

Most layers do their work with a matrix multiplication, which sounds scary and turns out to be a glorified weighted average. Let's build it.

2. The linear layer — matrix multiplication

The linear function is the workhorse of the entire model. Here is the complete Java code:

static Value[] linear(Value[] x, Value[][] w) {
    Value[] out = new Value[w.length];
    for (int i = 0; i < w.length; i++) {       // for each output slot
        out[i] = Value.of(0);
        for (int j = 0; j < x.length; j++)      // sum over all inputs
            out[i] = out[i].add(w[i][j].mul(x[j]));
    }
    return out;
}

Read it slowly. x is the input vector. w is a grid of weights (a matrix — a list of rows, each row a list of numbers). For each output slot i, we walk every input x[j], multiply it by the matching weight w[i][j], and add up the results. Each output is the weighted sum of all the inputs.

The voting analogy Think of each output as a committee vote. The committee looks at every input value, but it trusts some inputs more than others — those trust levels are the weights. A weight of 0 means "ignore this input entirely"; a large weight means "this input matters a lot to my decision"; a negative weight means "this input argues against my output". One row of the matrix = one committee = one output number. The whole point of training is to discover the right weights.

A tiny worked example

Let the input be x = [2, 3] and the weight matrix be:

w = [ [1, 0],  [0.5, 0.5] ]

Two rows → two outputs. Compute each as "row dotted with input":

  • out[0] = row [1, 0] → 1×2 + 0×3 = 2. (This committee only listens to the first input.)
  • out[1] = row [0.5, 0.5] → 0.5×2 + 0.5×3 = 2.5. (This one averages both inputs equally.)
linear([2, 3]) = [2, 2.5]

In microGPT, when you see linear(x, sd.get("layer0.attn_wq")), this exact loop runs on a 16×16 weight matrix: it reads a 16-element vector and produces a new 16-element vector, where each of the 16 outputs is a different weighted blend of all 16 inputs.

Concrete example

Input (3 numbers): x = [0.5, −0.3, 0.8]

Weight matrix (2 rows × 3 cols, so it maps 3D → 2D): [[0.1, 0.4, −0.2], [0.3, −0.1, 0.5]]

Output (each row dotted with the input):

out[0] = 0.1×0.5 + 0.4×(−0.3) + (−0.2)×0.8 = −0.23

out[1] = 0.3×0.5 + (−0.1)×(−0.3) + 0.5×0.8 = 0.58

Result: [−0.23, 0.58]. A 3-number representation became a 2-number one — the matrix sets both the blend and the output size.

Real-life analogy A linear layer is a translator. It converts information from one "language" (say, a 16-dimensional space) into another (a 64-dimensional space). Just like translating French to English, the meaning is preserved but the representation changes. The weight matrix is the "dictionary" that defines exactly how to translate one into the other — and training is the model writing its own dictionary.

3. ReLU — the activation function

Here is the entire ReLU layer in the transformer — one line:

for (int i = 0; i < x.length; i++)
    x[i] = x[i].relu();      // relu(v) = max(0, v)

ReLU ("Rectified Linear Unit") is almost laughably simple: keep positive numbers as they are, replace every negative number with zero. relu(3) = 3, relu(-3) = 0, relu(0) = 0. That's the whole function.

Concrete example

Input (4 numbers, say one slice of the hidden vector): [−0.23, 0.58, −1.2, 0.03]

Output: [0, 0.58, 0, 0.03]

Every negative number was clamped to 0; the positives passed through untouched. That single kink — flat for negatives, straight-through for positives — is the non-linearity that lets stacked layers learn curved patterns.

Real-life analogy ReLU is a bouncer at a club. It lets positive signals through unchanged but blocks all negative signals (sets them to zero) — like a bouncer who waves in anyone with a valid ID (positive) and turns away everyone without one (negative). This simple "gate" is what gives neural networks the ability to learn complex, non-linear patterns.

Why bother? Because without it, the network is fake

The crucial reason Stacking linear layers with nothing between them is pointless. Two matrix multiplications in a row can always be collapsed into a single matrix multiplication — mathematically, linear(linear(x)) equals linear'(x) for some other weights. So a hundred stacked linear layers would have exactly the same power as one. You need something non-linear in between to break that collapse and let the network represent complicated, curved relationships. ReLU is the simplest possible non-linearity that does the job.

"Non-linear" just means "not a straight line". The kink in ReLU at zero — straight along the bottom, then bending upward — is enough curvature that, when you stack many of them, the network can approximate almost any function. It is the bend that gives the model its expressive power.

But why ReLU specifically — why not something fancier? Because ReLU (max(0, x)) is the simplest thing that breaks the linear collapse, and simple wins here. It's trivial to compute and trivial to differentiate — the slope is just 1 when positive, 0 when negative. Fancier curves exist: GPT-2 used GeLU, a smoothed version, and the earliest networks used sigmoid. But sigmoid is slower and suffers from vanishing gradients, and for a toy model the extra polish buys nothing. ReLU does the one job a non-linearity must do, at the lowest possible cost.

Demo: the ReLU function — move the input, see it clamp at zero

Left of zero the line lies flat on the floor (output 0). Right of zero it rises straight up (output = input). The red dot is your value passing through the gate.

In microGPT, ReLU lives in the MLP block, applied to every element of the widened hidden vector. It is the model's only source of non-linearity in that block.

4. RMSNorm — keeping numbers civilized

As a vector flows through many layers, its numbers can drift — growing into the thousands or shrinking toward zero. Either extreme makes training unstable (gradients explode or vanish). Normalization rescales the vector back into a sane range at each step. microGPT uses RMSNorm:

static Value[] rmsnorm(Value[] x) {
    Value ms = Value.of(0);
    for (Value xi : x)
        ms = ms.add(xi.mul(xi));        // sum of squares
    ms = ms.div(Value.of(x.length));    // mean of squares
    Value scale = ms.add(Value.of(1e-5)).pow(-0.5);  // 1 / sqrt(ms)
    return Arrays.stream(x)
        .map(xi -> xi.mul(scale))       // divide every element by RMS
        .toArray(Value[]::new);
}

"RMS" means Root Mean Square — read it right-to-left: square every number, take the mean (average) of those squares, then take the square root. It is a measure of "how big, on average, are the numbers in this vector". RMSNorm divides every element by that single number, so the whole vector is scaled to a consistent overall size.

But why do we need normalization at all? Because every layer multiplies and adds, and those operations compound. After a few blocks a vector's numbers can balloon into the thousands (overflow) or collapse toward zero (underflow), and either way the gradients that depend on them explode or vanish — training stalls. RMSNorm is the volume control: after each block it rescales the vector back to a consistent magnitude, so the next block always receives numbers in a sane range. It resets the loudness without changing the tune.
Real-life analogy RMSNorm is the "normalize volume" feature on Spotify. Without it, some songs blast loud and others are barely audible; Spotify rescales every track to a consistent level so you're not constantly reaching for the volume knob. RMSNorm does the same for the vectors flowing through the network — keeping their magnitude steady so numbers never blast too loud or fade to silence as they pass through layer after layer.

Step by step with numbers

Take the input vector x = [4, 3]:

StepCalculationResult
Square each element4² = 16,  3² = 9[16, 9]
Mean of squares (ms)(16 + 9) / 212.5
Root → RMS√12.5≈ 3.54
Divide each element4 / 3.54,  3 / 3.54[1.13, 0.85]

Notice what survived and what changed. The raw sizes shrank from [4, 3] down to roughly [1.13, 0.85] — much tamer. But the relative proportions are preserved: 4 was bigger than 3, and 1.13 is still bigger than 0.85 by the same ratio. RMSNorm doesn't distort the shape of the information; it only fixes the scale. (The tiny 1e-5 in the code — "epsilon" — is just insurance so we never divide by zero on an all-zeros vector.)

Concrete example

Input: [3.0, −1.0, 2.0]

RMS = √((3² + (−1)² + 2²) / 3) = √(14/3) = √4.67 ≈ 2.16

Output (divide each by the RMS): [3.0/2.16, −1.0/2.16, 2.0/2.16] = [1.39, −0.46, 0.93]

The vector keeps its shape — 3 is still the biggest, the negative is still negative — but its overall magnitude is pulled back to about 1, so numbers can't snowball as the vector flows through layer after layer.

Demo: RMSNorm — type a vector, watch the rescale

Blue bars = input, green bars = after RMSNorm. Try huge values like 800, 600 — the output stays the same civilized shape as 4, 3 did.

RMSNorm vs. LayerNorm You may meet LayerNorm elsewhere (GPT-2 used it). It does the same job but first subtracts the mean of the vector, then divides by the standard deviation. RMSNorm skips the mean-subtraction step — it just divides by the RMS. Simpler, fewer operations, and in practice works just as well. Same idea, leaner code. microGPT chose the simpler one.

5. Putting layers together: the MLP block

Now watch real layers compose. The "MLP block" (Multi-Layer Perceptron) is one of the two engines inside a transformer, and it is built entirely from the three layers above:

x = rmsnorm(x);                          // normalize
x = linear(x, sd.get("layer0.mlp_fc1")); // expand: 16 → 64
for (int i = 0; i < x.length; i++)
    x[i] = x[i].relu();                  // non-linearity
x = linear(x, sd.get("layer0.mlp_fc2")); // compress: 64 → 16

Four steps, and you already understand all four:

  1. Normalize with RMSNorm so the numbers start in a sane range.
  2. Expand with a linear layer: a 16×64 weight matrix turns 16 numbers into 64. (4 * nEmbd = 64 in the code.)
  3. ReLU every one of those 64 numbers — the non-linearity that gives the block its power.
  4. Compress with another linear layer: a 64×16 matrix squeezes back down to 16.
16  → (expand) →  64  → (ReLU) →  64  → (compress) →  16
Why expand then compress? The wide middle layer (64 slots) gives the network extra thinking room. With only 16 slots there isn't space to disentangle complex patterns; blowing up to 64, applying the non-linearity there, then folding back down to 16 lets the block compute a much richer transformation than its narrow input and output sizes would suggest. The 64-wide layer is scratch paper the model does its rough work on before writing the clean 16-number answer.
Real-life analogy The MLP is brainstorming, then summarizing. Expanding 16 → 64 is the brainstorm — spread out and generate lots of rough ideas. ReLU is the filter — throw out the bad ideas. Compressing 64 → 16 is the summary — fold everything back down into a few key insights. The wide middle layer simply gives the network more room to "think" before condensing back to a compact answer.

6. Residual connections — "describe the change, not the whole thing"

One last piece glues the blocks together. After the MLP (and after attention), microGPT does this:

for (int i = 0; i < nEmbd; i++)
    x[i] = x[i].add(xr[i]);   // add the block's output back to its input

Here xr is the vector as it was before the block ran, saved aside. Instead of replacing the input with the block's output, we add them. The block therefore only has to learn a small correction to apply to what came in — not reconstruct the whole vector from scratch. This is called a residual connection (or "skip connection").

The directions analogy Suppose you're guiding someone to a destination. You could re-describe their exact position from absolute zero every single step — exhausting and error-prone. Or you could just say "take two steps left" — a small correction relative to where they already are. Residual connections are the second style: each block says "here's a small nudge to add", and the running vector x carries the accumulated position forward.
Why it matters for learning Residuals also keep gradients healthy. Because the input xr is added straight through, the add operation passes gradient back to xr with a local grad of 1 (remember from the autograd chapter: add has gradient 1 to each child). That creates an unbroken "gradient highway" from the loss all the way back to the early layers, so deep stacks of blocks can still learn. Without residuals, gradients shrink as they travel back through many layers and the early layers barely train — the infamous vanishing gradient problem.

Recap

What you now know
  • A layer is a function: vector in → vector out.
  • linear is matrix multiplication — each output is a weighted vote over all inputs. The weights are what training adjusts.
  • ReLU = max(0, x): the simplest non-linearity, and without it stacked linear layers collapse into one.
  • RMSNorm divides a vector by its root-mean-square, taming the scale while preserving proportions, so training stays stable.
  • The MLP block chains them: normalize → expand 16→64 → ReLU → compress 64→16.
  • Residual connections add the block's output back to its input, so each block learns a small correction and gradients flow cleanly to the bottom.

These are the generic building blocks. The next chapter introduces the one layer that makes a transformer a transformer — attention — which lets each position in the sequence look at the others.
