2The Value record — the autograd engine

    // Autograd engine for recursive chain rule application
    // through a computation graph
    record Value(double[] data, double[] grad,
                 Value[] children, double[] localGrads) {

        Value(double v) {
            this(new double[]{v}, new double[]{0},
                 new Value[0], new double[0]);
        }

        static Value of(double v) {
            return new Value(v);
        }

        double d() { return data[0]; }
        double g() { return grad[0]; }
        void addGrad(double v) { grad[0] += v; }

        static Value op(double val, Value[] ch, double[] lg) {
            return new Value(new double[]{val},
                             new double[]{0}, ch, lg);
        }

        Value add(Value o) {
            return op(d() + o.d(),
                      new Value[]{this, o},
                      new double[]{1, 1});
        }

        Value mul(Value o) {
            return op(d() * o.d(),
                      new Value[]{this, o},
                      new double[]{o.d(), d()});
        }

        Value pow(double e) {
            return op(Math.pow(d(), e),
                      new Value[]{this},
                      new double[]{e * Math.pow(d(), e - 1)});
        }

        Value log() {
            return op(Math.log(d()),
                      new Value[]{this},
                      new double[]{1 / d()});
        }

        Value exp() {
            double e = Math.exp(d());
            return op(e, new Value[]{this}, new double[]{e});
        }

        Value relu() {
            return op(Math.max(0, d()),
                      new Value[]{this},
                      new double[]{d() > 0 ? 1 : 0});
        }

        Value neg() { return mul(of(-1)); }
        Value sub(Value o) { return add(o.neg()); }
        Value div(Value o) { return mul(o.pow(-1)); }

        void backward() {
            List<Value> topo = new ArrayList<>();
            Set<Value> visited = new HashSet<>();
            buildTopo(this, topo, visited);
            grad[0] = 1;
            for (int i = topo.size() - 1; i >= 0; i--) {
                Value v = topo.get(i);
                for (int j = 0; j < v.children.length; j++)
                    v.children[j].addGrad(
                        v.localGrads[j] * v.grad[0]);
            }
        }

        static void buildTopo(Value v, List<Value> topo,
                               Set<Value> visited) {
            if (visited.add(v)) {
                for (Value c : v.children)
                    buildTopo(c, topo, visited);
                topo.add(v);
            }
        }
    }
This is Chapter 5 made concrete. Each Value holds its number (data), its gradient (grad), and pointers to the children it was built from plus the local derivative w.r.t. each child (localGrads). Every operation (add, mul, pow, log, exp, relu) records those links, silently building a computation graph. backward() then topologically sorts the graph and walks it in reverse, multiplying local gradients along the way — the chain rule, automated.

class Value:
    def __init__(self, data, _children=()):
        self.data = data
        self.grad = 0.0
        self._prev = set(_children)
        self._backward = lambda: None
    def __add__(self, other):
        out = Value(self.data + other.data, (self, other))
        def _backward():
            self.grad += out.grad
            other.grad += out.grad
        out._backward = _backward
        return out

Ch08 · The Transformer
But why new Random(42) — why a fixed seed? Reproducibility. A fixed seed makes the random number generator produce the same sequence every run, so the weights initialize identically and the shuffle orders the names identically — you get the same training result each time. The number 42 itself is arbitrary (a Hitchhiker's Guide joke); any fixed value works. In research, a pinned seed is what lets you change one thing and trust that any difference in the outcome came from that change, not from fresh randomness.

4Parameter initialization — matrix()

    static Value[][] matrix(int nout, int nin) {
        Value[][] m = new Value[nout][nin];
        for (int i = 0; i < nout; i++)
            for (int j = 0; j < nin; j++) {
                m[i][j] = Value.of(rng.nextGaussian() * 0.08);
                params.add(m[i][j]);
            }
        return m;
    }
Every weight in the model is born here. matrix(nout, nin) allocates an nout × nin grid of Values, each a small random Gaussian number (× 0.08, so they start near zero), and — crucially — registers each one in the global params list so the optimizer can later update it. Below, main() calls matrix() to fill the sd (“state dict”) map with every named weight matrix: token embeddings wte, position embeddings wpe, the attention/MLP weights per layer, and the final lm_head. Total: 4,192 numbers.
Ch03 · Embeddings
But why Gaussian random init with std = 0.08? The standard deviation sets how spread out the starting weights are, and both extremes break training. Too large (say 1.0) and the first predictions are wild, gradients explode, and the model diverges. Too small (say 0.001) and every weight is essentially zero — the model does almost nothing and learns at a crawl. 0.08 is the sweet spot: large enough to break symmetry (so neurons start out different), small enough to stay stable. It's no coincidence that 0.08 ≈ 1/√nEmbd, a common scaling heuristic — the right value depends on layer size.

5linear() — matrix × vector

    static Value[] linear(Value[] x, Value[][] w) {
        Value[] out = new Value[w.length];
        for (int i = 0; i < w.length; i++) {
            out[i] = Value.of(0);
            for (int j = 0; j < x.length; j++)
                out[i] = out[i].add(w[i][j].mul(x[j]));
        }
        return out;
    }
The workhorse of every neural network: take an input vector x and a weight matrix w, and produce one output number per row by computing a dot product (Chapter 2) of that row with x. Every dense connection in the model — attention's Q/K/V projections, the MLP, the output head — is just a call to this. It builds Value nodes as it goes, so gradients flow through automatically.

def linear(x, w):
    # one output per row of w
    return [sum(wij * xj for wij, xj in zip(row, x))
            for row in w]

6softmax() — logits → probabilities

    static Value[] softmax(Value[] logits) {
        double max = Arrays.stream(logits)
            .mapToDouble(Value::d).max().getAsDouble();
        Value[] exps = Arrays.stream(logits)
            .map(v -> v.sub(Value.of(max)).exp())
            .toArray(Value[]::new);
        Value total = Arrays.stream(exps)
            .reduce(Value.of(0), Value::add);
        return Arrays.stream(exps)
            .map(e -> e.div(total))
            .toArray(Value[]::new);
    }
Turns a list of raw scores (logits) into a probability distribution that sums to 1 (Chapter 4). It exponentiates each score and divides by the total. The max subtraction is a numerical-stability trick — it doesn't change the result (the same constant cancels top and bottom) but stops exp() from overflowing on large logits. Used both inside attention (to weight the heads) and at the very end (to score the next token).

Python (Karpathy's original)

def softmax(logits):
    m = max(v.data for v in logits)
    exps = [(v - m).exp() for v in logits]
    s = sum(exps)
    return [e / s for e in exps]

7rmsnorm() — keeping numbers tame

    static Value[] rmsnorm(Value[] x) {
        Value ms = Value.of(0);
        for (Value xi : x)
            ms = ms.add(xi.mul(xi));
        ms = ms.div(Value.of(x.length));
        Value scale = ms.add(Value.of(1e-5)).pow(-0.5);
        return Arrays.stream(x)
            .map(xi -> xi.mul(scale))
            .toArray(Value[]::new);
    }
Normalization. As vectors pass through layers their magnitudes can drift, making training unstable. RMSNorm rescales a vector so its root-mean-square size is ~1: compute the mean of the squares, take (ms + 1e-5)−0.5 (one over the square root), and multiply every component by it. The 1e-5 avoids dividing by zero. It's applied before attention, before the MLP, and once at the start.

8gpt() — the transformer forward pass

    static Value[] gpt(int tokenId, int posId,
                       List<Value[]>[] keys,
                       List<Value[]>[] values) {
        Value[] x = new Value[nEmbd];
        for (int i = 0; i < nEmbd; i++)
            x[i] = sd.get("wte")[tokenId][i]
                .add(sd.get("wpe")[posId][i]);

        x = rmsnorm(x);

        for (int li = 0; li < nLayer; li++) {
            // 1) Multi-head Attention block
            Value[] xr = x;
            x = rmsnorm(x);
            Value[] q = linear(x,
                sd.get("layer" + li + ".attn_wq")),
                k = linear(x,
                    sd.get("layer" + li + ".attn_wk")),
                v = linear(x,
                    sd.get("layer" + li + ".attn_wv"));
            keys[li].add(k);
            values[li].add(v);

            Value[] xAttn = new Value[0];
            for (int h = 0; h < nHead; h++) {
                int hs = h * headDim;
                Value[] qh = Arrays.copyOfRange(
                    q, hs, hs + headDim);
                int T = keys[li].size();
                Value[] attnLogits = new Value[T];
                for (int t = 0; t < T; t++) {
                    attnLogits[t] = Value.of(0);
                    for (int j = 0; j < headDim; j++)
                        attnLogits[t] = attnLogits[t].add(
                            qh[j].mul(
                                keys[li].get(t)[hs + j]));
                    attnLogits[t] = attnLogits[t].div(
                        Value.of(Math.sqrt(headDim)));
                }
                Value[] aw = softmax(attnLogits);
                Value[] ho = new Value[headDim];
                for (int j = 0; j < headDim; j++) {
                    ho[j] = Value.of(0);
                    for (int t = 0; t < T; t++)
                        ho[j] = ho[j].add(
                            aw[t].mul(
                                values[li].get(t)[hs + j]));
                }
                xAttn = concat(xAttn, ho);
            }

            x = linear(xAttn,
                sd.get("layer" + li + ".attn_wo"));
            for (int i = 0; i < nEmbd; i++)
                x[i] = x[i].add(xr[i]);

            // 2) MLP block
            xr = x;
            x = rmsnorm(x);
            x = linear(x,
                sd.get("layer" + li + ".mlp_fc1"));
            for (int i = 0; i < x.length; i++)
                x[i] = x[i].relu();
            x = linear(x,
                sd.get("layer" + li + ".mlp_fc2"));
            for (int i = 0; i < nEmbd; i++)
                x[i] = x[i].add(xr[i]);
        }

        return linear(x, sd.get("lm_head"));
    }

    static Value[] concat(Value[] a, Value[] b) {
        Value[] c = new Value[a.length + b.length];
        System.arraycopy(a, 0, c, 0, a.length);
        System.arraycopy(b, 0, c, a.length, b.length);
        return c;
    }
The heart of the model, and the payoff of Chapters 7 and 8. For one token at one position it: (1) looks up the token embedding wte and adds the position embedding wpe; (2) runs the attention block — projects to query/key/value, caches K and V, and for each of the 4 heads computes scaled dot-product attention over all positions so far (the causal cache is what makes it look only backwards), then concatenates the heads and projects out; (3) runs the MLP block — linear → relu → linear; with a residual (x + ...) added around both blocks. Finally lm_head projects the 16-vector to 27 logits — one score per possible next token. concat() below just glues the per-head outputs back together.

Ch01 · Tokens & Vocabulary
But why download names from the internet? The model has nothing to learn from until you give it examples. Karpathy's names.txt is ~32,000 real first names (emma, olivia, ava, …), and the statistical patterns inside them — which letters follow which, how names tend to start and end, typical lengths — are exactly what the model absorbs during training. No data, nothing to imitate. The download just fetches that raw material once and caches it in input.txt.
But why shuffle the training data (Collections.shuffle)? Without shuffling, the model would see names in sorted order: every a-name, then every b-name, and so on. That creates bias — after 500 a-names in a row it tilts toward a-name patterns and temporarily "forgets" about z-names. Shuffling means each step sees a random mix, so learning stays balanced across the whole alphabet instead of lurching from one cluster to the next.

10main() — the training loop

        // Adam optimizer setup and training loop
        int nSteps = 1000;
        double lr = 0.01, b1 = 0.85, b2 = 0.99,
               epsAdam = 1e-8;
        double[] m = new double[params.size()],
                 v = new double[params.size()];

        for (int step = 0; step < nSteps; step++) {
            String doc = docs.get(step % docs.size());
            List<Integer> tokens = new ArrayList<>();
            tokens.add(BOS);
            for (char c : doc.toCharArray())
                tokens.add(uchars.indexOf(c));
            tokens.add(BOS);

            // Forward pass building the computation graph
            int n = Math.min(blockSize,
                             tokens.size() - 1);
            List<Value[]>[] keys = new List[nLayer],
                            vals = new List[nLayer];
            for (int i = 0; i < nLayer; i++) {
                keys[i] = new ArrayList<>();
                vals[i] = new ArrayList<>();
            }

            Value loss = Value.of(0);
            for (int pos = 0; pos < n; pos++) {
                Value[] logits = gpt(tokens.get(pos),
                                     pos, keys, vals);
                Value[] probs = softmax(logits);
                loss = loss.add(
                    probs[tokens.get(pos + 1)]
                        .log().neg());
            }
            loss = loss.div(Value.of(n));

            // Backward pass for gradient computation
            loss.backward();

            // Adam parameter update with linear LR decay
            double lrt = lr * (1.0 - step / (double) nSteps);
            for (int i = 0; i < params.size(); i++) {
                double g = params.get(i).g();
                m[i] = b1 * m[i] + (1 - b1) * g;
                v[i] = b2 * v[i] + (1 - b2) * g * g;
                double mh = m[i]
                    / (1 - Math.pow(b1, step + 1)),
                    vh = v[i]
                    / (1 - Math.pow(b2, step + 1));
                params.get(i).data[0] -=
                    lrt * mh / (Math.sqrt(vh) + epsAdam);
                params.get(i).grad[0] = 0;
            }

            System.out.printf(
                "\rstep %4d/%4d | loss %.4f",
                step + 1, nSteps, loss.d());
        }
Chapter 9 in code. Set up Adam's state (m, v arrays). For each of 1,000 steps: take a name, wrap it in BOS markers and tokenize; run the forward pass across positions, accumulating cross-entropy loss (−log of the correct token's probability); call loss.backward() to fill every gradient; then the Adam update steps all 4,192 parameters downhill with a linearly-decaying learning rate, and zeroes the gradients. The \r in the printf redraws the loss in place so you watch it fall live.

Read those three loss numbers as the whole story of training. Step 1's 3.40 is essentially −ln(1/27) — the model is guessing uniformly among 27 characters. By step 500 it's down to 2.31, and by step 1000 2.19: the model now puts real probability on the letters that actually tend to follow, so it's "surprised" by the next character far less often. That 3.40 → 2.19 drop is the learning — no rule about names was ever written down.

Real-life analogy The whole training process is a child learning to write names by hand. Show a child who has never seen a name 32,000 name cards, one at a time. At first they scribble randomly. But gradually they notice patterns: names often start with consonants, vowels appear frequently, certain letter pairs are common ("th", "sh", "an"). After working through all 32,000 cards (1,000 training steps over batches), they can invent plausible new names they've never seen. That's exactly what this loop does — just with numbers instead of handwriting.

                    }
                }
                if (tokenId == BOS) break;
                sb.append(uchars.get(tokenId));
            }
            System.out.printf("sample %2d: %s%n",
                              s + 1, sb);
        }
    }
The fun part. With training done, generate 20 brand-new names. Start from BOS, run gpt() to get logits, divide them by temp = 0.5 (temperature < 1 sharpens the distribution toward confident picks — Chapter 4), softmax to probabilities, then sample: draw a random number and walk the cumulative probabilities to pick a token. Append its character and feed it back as the next input. Stop when the model emits BOS (“name finished”) or we hit blockSize. These names exist nowhere in the training data — the model invented them.
Ch04 · Temperature & Sampling
Concrete example

Let's hand-generate one name, character by character. (Probabilities below are illustrative — the real ones come from the trained weights.)

  1. Start with tokenId = BOS, position 0.
  2. gpt(BOS, 0) → 27 raw logits. Divide each by temp = 0.5 (this sharpens the distribution — dividing by a number < 1 stretches the gaps), then softmax:
    {'a':0.18, 'm':0.12, 's':0.09, …}. Draw a random number, walk the cumulative probabilities → lands on m.
  3. Feed m back: gpt('m', 1) → softmax → {'a':0.52, 'e':0.11, 'i':0.08, …} → sample → a.
  4. gpt('a', 2) → {'r':0.21, 'n':0.14, …} → r.
  5. gpt('r', 3) → {'a':0.40, …} → a.
  6. gpt('a', 4) → this time the sample lands on BOS → the name is finished.

Output: m → a → r → a = "mara" — a name the model invented, character by character, that never appears verbatim in any single training step's objective. Lowering temp makes it pick the popular letters more often (safer, more repetitive names); raising it flattens the distribution (weirder, more varied names).

Running it yourself

No build tools, no dependencies. You need a JDK (Java 17 or newer) on your path. With the source saved as GPT.java (drop the package line on the first line if you want the simplest possible compile), from that folder:

# 1. compile — produces GPT.class javac GPT.java # 2. run — trains for ~1000 steps, then generates 20 names java GPT

The first run downloads names.txt (~228 KB) into input.txt, then trains for a couple of minutes on a single CPU core. Expected output:

num docs: 32033 vocab size: 27 num params: 4192 step 1/1000 | loss 3.4011 step 500/1000 | loss 2.3110 step 1000/1000 | loss 2.1903 --- inference (new, hallucinated names) --- sample 1: shade sample 2: jara sample 3: amara ...
